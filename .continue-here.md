### Fix 1: Drain window not activating when TTS pre-enabled (index.jsx)
- Old code: drain only activated when `pttForcedTtsRef.current && !prevTtsEnabledRef.current`
- New code: drain always activates when `narrationDeviceName` is set; TTS disable is guarded by `wasForced`

### Fix 2: Grace period too short (index.jsx)
- Old: 8s grace cap — STT + ElevenLabs synthesis can easily exceed 8s
- New: 30s hard cap, but drain exits EARLY once translation arrives AND TTS finishes
- Condition changed: `(pttAwaitingTranslationRef.current && withinHardCap)` instead of the old `(... && stillWithinGrace) || stillWithinGrace`
- File: `src/window/Monitor/index.jsx` — `finishWhenDrained` inside `handlePttEnd()`

### Fix 3: Race condition — refs set AFTER await (index.jsx) ← CRITICAL, last fix
- Old: `prevTtsEnabledRef.current` and `pttForcedTtsRef.current` set AFTER `await restartCaptureWithSource()`

- New: Both refs are set BEFORE the await
- File: `src/window/Monitor/index.jsx` — `handlePttStart()`

<decisions_made>

- Drain window uses `pttAwaitingTranslationRef` as the PRIMARY signal (not a fixed time window)
- 30s hard cap is the absolute maximum to prevent infinite wait if STT completely fails
- `wasForced` determines whether TTS should be disabled after drain (only if PTT forced it on)
- TTS plays locally (speakers) AND injects to BlackHole simultaneously — muting local playback deferred
- `prevTtsEnabledRef` + `pttForcedTtsRef` must be set BEFORE any await in handlePttStart
